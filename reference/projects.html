Position

(주)유머스트알엔디 / 인턴사원

Overview

서울특별시 도봉구의 지역상권 활성화를 위한 관광 정보 안내 서비스입니다. 스토리텔링 기반 여행 코스, 맛집 정보, 지역 소상공인 연계 할인 쿠폰 발행/사용 등의 기능을 제공하며, 현재 플레이스토어와 앱스토어에 출시되어 다운로드 가능합니다.

Key Skills
AWS Terraform Docker Compose GitHub Actions CloudWatch SSM Nginx

Role

V1 — AWS 인프라를 설계, 구축하고 운영하는 업무를 담당하였습니다.

  • 초기 애플리케이션의 백엔드 인프라를 설계
  • GitHub Actions를 사용하여 SSH와 Docker Compose 기반의 CI/CD 파이프라인 구축

V2 — 배포 자동화 및 운영 안정성 개선을 중심으로 인프라를 고도화하였습니다.

  • Terraform을 도입하여 인프라를 코드로 관리
  • GitHub Actions의 OIDC를 활용하여 IAM Role 기반 인증 구조 구성
  • SSM을 통해 EC2 접근 및 배포를 수행, SSH Key 관리 없이 보안성 개선
  • CloudWatch를 활용하여 애플리케이션 로그 수집 및 모니터링 환경 구축
V1

초기 인프라 설계 및 구축

단일 EC2 + RDS 기반 AWS 설계 및 배포

아키텍처 설계

도봉라이프의 초기 버전에서는 개발 환경 중심으로 최소 비용과 빠른 배포가 요구되었습니다. 따라서 단일 EC2 기반으로 Nginx, Spring Boot, Redis를 Docker Compose로 통합 배포하였습니다.

VPC를 구성하고 퍼블릭 서브넷(EC2)과 프라이빗 서브넷(RDS)을 분리하여 외부 접근은 Nginx를 통해서만 가능하도록 설계하였습니다. Security Group을 통해 애플리케이션 계층 간 접근을 제한하여 데이터베이스를 외부로부터 격리하였습니다. Route53과 S3를 연계하여 도메인 및 정적 리소스 접근 구조를 구성하였습니다.

CI/CD 파이프라인

수동 배포를 자동화하기 위해 GitHub Actions를 활용하여 CI/CD 파이프라인을 구축하였습니다. 코드 푸시나 PR 머지 시 GitHub Actions가 트리거되어 빌드 및 테스트를 수행하고, 빌드된 이미지를 Docker Hub에 푸시하는 작업을 구성하였습니다. 이후 SSH를 통해 EC2에 접속하여 Docker Compose 명령어로 애플리케이션을 재배포하는 구조입니다.

V2

인프라 고도화 및 운영 안정성 개선

역할 기반 아키텍처 설계 및 Terraform 도입

아키텍처 전환

웹 어드민 페이지를 개발하면서 FE 서버를 추가 배포하였습니다. 트래픽 증가에 대비하여 인프라 구조 개선과 운영 로그 수집을 위한 모니터링 시스템을 구축하였습니다.

단일 서버 구조에서 역할 기반 아키텍처로 전환하였습니다. Nginx, Backend, Frontend 서버를 각각 독립된 EC2로 분리하고, 외부 노출 영역(Nginx: 퍼블릭 서브넷)과 내부 서비스 영역(FE/BE: 프라이빗 서브넷)을 분리하여 보안 수준을 향상시켰습니다.

  • Redis를 EC2에서 Amazon ElastiCache로 이전
  • SSH(22번 포트) 기반 접근 제거, AWS Systems Manager 접근 방식으로 전환
  • GitHub Actions + OIDC 기반으로 CD 재구성
  • Terraform을 활용하여 VPC / Subnet / EC2 / RDS / IAM 등 리소스 코드화
  • AWS Systems Manager Parameter Store로 환경 변수 및 민감 정보 관리
  • user-data 파일을 이용하여 EC2 기본 설정 자동화
  • Amazon CloudWatch를 활용하여 EC2 로그 수집 및 애플리케이션 로그 확인
